## 1. Supabase-Projekt erstellen

### Schritt 1.1: Account erstellen

1. Gehen Sie zu: **https://supabase.com**
2. Klicken Sie auf **"Start your project"** oder **"Sign Up"**
3. Registrieren Sie sich mit:
   - GitHub Account (empfohlen)
   - ODER E-Mail-Adresse

### Schritt 1.2: Neues Projekt erstellen

1. Nach dem Login klicken Sie auf **"New Project"**
2. Füllen Sie das Formular aus:
   - **Name**: `Invox` (oder beliebiger Name)
   - **Database Password**: Wählen Sie ein sicheres Passwort (WICHTIG: Notieren Sie sich dieses!)
   - **Region**: Wählen Sie die nächstgelegene Region (z.B. `Europe (Frankfurt)`)
   - **Pricing Plan**: Wählen Sie `Free` für den Start

3. Klicken Sie auf **"Create new project"**
4. ⏳ Warten Sie 1-2 Minuten, bis das Projekt erstellt ist

## 3. Datenbank einrichten

Invox verwendet einen **Key-Value Store** für flexible Datenspeicherung. Wir müssen die Deno KV Tabelle erstellen.

### Schritt 3.1: SQL Editor öffnen

1. Klicken Sie in der linken Sidebar auf **"SQL Editor"**
2. Klicken Sie auf **"New Query"**

### Schritt 3.2: KV-Store Tabelle erstellen

Kopieren Sie folgenden SQL-Code und führen Sie ihn aus:

```sql
-- Erstelle KV Store Tabelle (WICHTIG: Name muss "kv_store_e5b0fe50" sein!)
CREATE TABLE IF NOT EXISTS kv_store_e5b0fe50 (
  key TEXT PRIMARY KEY,
  value JSONB NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Index für schnellere Prefix-Suchen
CREATE INDEX IF NOT EXISTS idx_kv_store_key_prefix ON kv_store_e5b0fe50(key text_pattern_ops);

-- Funktion für automatisches updated_at
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ language 'plpgsql';

-- Trigger für updated_at
CREATE TRIGGER update_kv_store_updated_at 
BEFORE UPDATE ON kv_store_e5b0fe50 
FOR EACH ROW 
EXECUTE FUNCTION update_updated_at_column();
```

Klicken Sie auf **"Run"** (rechts unten)

✅ Sie sollten eine Erfolgsmeldung sehen: `Success. No rows returned`

### Schritt 3.3: Row Level Security (RLS) aktivieren (Optional aber empfohlen)

Für Multi-User-Szenarien sollten Sie RLS aktivieren:

```sql
-- RLS aktivieren
ALTER TABLE kv_store_e5b0fe50 ENABLE ROW LEVEL SECURITY;

-- Policy: Benutzer können nur ihre eigenen Daten sehen
CREATE POLICY "Users can view their own data"
  ON kv_store_e5b0fe50
  FOR SELECT
  USING (
    -- Wenn key mit "settings:" beginnt, muss user_id im key enthalten sein
    CASE 
      WHEN key LIKE 'settings:%' THEN 
        key LIKE '%' || auth.uid()::text || '%'
      ELSE 
        true
    END
  );

-- Policy: Benutzer können nur ihre eigenen Daten einfügen
CREATE POLICY "Users can insert their own data"
  ON kv_store_e5b0fe50
  FOR INSERT
  WITH CHECK (
    CASE 
      WHEN key LIKE 'settings:%' THEN 
        key LIKE '%' || auth.uid()::text || '%'
      ELSE 
        true
    END
  );

-- Policy: Benutzer können nur ihre eigenen Daten aktualisieren
CREATE POLICY "Users can update their own data"
  ON kv_store_e5b0fe50
  FOR UPDATE
  USING (
    CASE 
      WHEN key LIKE 'settings:%' THEN 
        key LIKE '%' || auth.uid()::text || '%'
      ELSE 
        true
    END
  );

-- Policy: Benutzer können nur ihre eigenen Daten löschen
CREATE POLICY "Users can delete their own data"
  ON kv_store_e5b0fe50
  FOR DELETE
  USING (
    CASE 
      WHEN key LIKE 'settings:%' THEN 
        key LIKE '%' || auth.uid()::text || '%'
      ELSE 
        true
    END
  );
```

## 4. Edge Functions deployen

Die Invox-App benötigt eine Edge Function für den KV-Store-Server.

### Schritt 4.1: Supabase CLI installieren

**Für macOS/Linux:**
```bash
brew install supabase/tap/supabase
```

**Für Windows:**
```bash
# Mit Scoop
scoop bucket add supabase https://github.com/supabase/scoop-bucket.git
scoop install supabase
```

**Oder mit NPM (alle Plattformen):**
```bash
npm install -g supabase
```

### Schritt 4.2: Supabase CLI login

```bash
supabase login
```

Dies öffnet Ihren Browser - melden Sie sich an und autorisieren Sie die CLI.

### Schritt 4.3: Projekt-Link erstellen

Im Invox-App Verzeichnis:

```bash
# Navigieren Sie zum Projekt-Ordner
cd /pfad/zu/invox

# Link zum Supabase-Projekt erstellen
supabase link --project-ref IHR-PROJECT-ID
```

Ersetzen Sie `IHR-PROJECT-ID` mit Ihrer Project ID (z.B. `lwqitbvbvhvdvpntqdgw`)

Wenn nach dem Database Password gefragt wird, geben Sie das Passwort aus Schritt 1.2 ein.

### Schritt 4.4: Edge Functions deployen

```bash
# Edge Function deployen
supabase functions deploy make-server-e5b0fe50
```

Wenn Sie nach Secrets gefragt werden:

```bash
# Service Role Key als Secret setzen (WICHTIG!)
supabase secrets set SUPABASE_SERVICE_ROLE_KEY="IHR-SERVICE-ROLE-KEY-HIER"
```

✅ Erfolgsmeldung: `Deployed Function make-server-e5b0fe50`

### Schritt 4.5: Funktion testen

```bash
# Test-Request
curl https://IHR-PROJECT-ID.supabase.co/functions/v1/make-server-e5b0fe50/health
```

Erwartete Antwort:
```json
{"status":"ok"}
```

## 8. Fehlerbehebung

### Problem 1: "Failed to fetch" Fehler

**Ursache**: CORS oder falsche URL

**Lösung**:
1. Prüfen Sie die `projectId` in `/utils/supabase/info.tsx`
2. Aktivieren Sie CORS in Supabase Dashboard (siehe Schritt 6.3)
3. Stellen Sie sicher, dass Edge Function deployed ist

### Problem 2: "Invalid API Key"

**Ursache**: Falscher oder abgelaufener API Key

**Lösung**:
1. Kopieren Sie den `anon public` Key erneut aus Supabase Dashboard
2. Aktualisieren Sie `/utils/supabase/info.tsx`
3. Starten Sie die App neu

### Problem 3: "Email not confirmed"

**Ursache**: Email-Bestätigung ist aktiviert

**Lösung**:
1. Prüfen Sie Ihr E-Mail-Postfach für Bestätigungs-E-Mail
2. ODER deaktivieren Sie "Enable email confirmations" in Auth Settings (nur für Entwicklung!)

### Problem 4: Edge Function Fehler

**Ursache**: Edge Function nicht deployed oder Fehler im Code

**Lösung**:
```bash
# Logs anschauen
supabase functions serve make-server-e5b0fe50

# Erneut deployen
supabase functions deploy make-server-e5b0fe50
```

### Problem 5: "Row Level Security" Fehler

**Ursache**: RLS blockiert Zugriff

**Lösung**:
```sql
-- Temporär RLS deaktivieren (nur für Tests!)
ALTER TABLE kv_store_e5b0fe50 DISABLE ROW LEVEL SECURITY;

-- Oder: Policies überprüfen
SELECT * FROM pg_policies WHERE tablename = 'kv_store_e5b0fe50';
```

### Problem 6: Daten werden nicht gespeichert

**Prüfen Sie:**

1. **Browser Console**: Gibt es JavaScript-Fehler?
2. **Network Tab**: Sehen Sie die API-Requests?
3. **Supabase Logs**: 
   - Dashboard → Logs → Function Logs
   - Sehen Sie Fehler?

### Problem 7: "Database password" vergessen

**Lösung**:
1. Gehen Sie zu Supabase Dashboard
2. Settings → Database → Reset Database Password
3. ⚠️ WARNUNG: Alle bestehenden Verbindungen werden getrennt!

### Problem 8: Edge Function kann nicht deployed werden

**Ursache**: Keine Supabase CLI oder Deployment-Probleme

**Alternative Lösung - Direkter Datenbank-Zugriff**:

Wenn Sie die Edge Functions nicht deployen können, können Sie direkt auf die Datenbank zugreifen:

1. Erstellen Sie eine neue Datei `/src/lib/supabase-direct.ts`:

```typescript
import { supabase } from './supabase';

// Direkter Zugriff auf kv_store_e5b0fe50 Tabelle
export const dbHelpersDirect = {
  get: async (key: string) => {
    const { data, error } = await supabase
      .from('kv_store_e5b0fe50')
      .select('value')
      .eq('key', key)
      .maybeSingle();
    
    if (error) {
      console.error('Error getting data:', error);
      return { data: null, error };
    }
    
    return { data: data?.value || null, error: null };
  },

  set: async (key: string, value: any) => {
    const { error } = await supabase
      .from('kv_store_e5b0fe50')
      .upsert({ key, value });
    
    if (error) {
      console.error('Error setting data:', error);
      return { data: null, error };
    }
    
    return { data: { key, value }, error: null };
  },

  delete: async (key: string) => {
    const { error } = await supabase
      .from('kv_store_e5b0fe50')
      .delete()
      .eq('key', key);
    
    if (error) {
      console.error('Error deleting data:', error);
      return { data: null, error };
    }
    
    return { data: { deleted: true, key }, error: null };
  },

  getAll: async (prefix?: string) => {
    let query = supabase.from('kv_store_e5b0fe50').select('key, value');
    
    if (prefix) {
      query = query.like('key', `${prefix}%`);
    }
    
    const { data, error } = await query;
    
    if (error) {
      console.error('Error getting all data:', error);
      return { data: null, error };
    }
    
    return { data: data?.map(d => d.value) || [], error: null };
  },
};
```

2. In `/src/lib/supabase.ts`, ersetzen Sie die `dbHelpers` Exporte durch:

```typescript
import { dbHelpersDirect } from './supabase-direct';
export const dbHelpers = dbHelpersDirect;
```

3. Stellen Sie sicher, dass die Tabelle `kv_store_e5b0fe50` existiert (siehe Schritt 3.2)

4. Starten Sie die App neu

⚠️ **Hinweis**: Diese Methode funktioniert, aber Edge Functions bieten bessere Performance und Skalierbarkeit.
